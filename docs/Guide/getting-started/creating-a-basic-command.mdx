In order to start registering commands you need to create a folder called `commands` (lowercase!) in the same directory as your "main" file. If you were following [Getting Started](./getting-started), that would be in your `src` directory.

Normally, when creating commands, you would define one command per file. However, as we support various forms of exporting commands from the file (`export class ...`, `export default class`, `module.exports = class ...` and `exports.MyCommand = class ...`) you can theoretically export multiple commands from the same file. Note that this does not make subcommands. We'll cover those in [Plugin Subcommands](../plugins/plugin-subcommands)

Create a `ping.js` file in your `commands` folder, which will send a message and then edit it with the elapsed time. Arguments and other features are covered in other pages.

## Creating a command class

Let's start by creating the file for the command class. In your `commands` folder, create a file named `ping.js` (or `ping.mjs` / `ping.ts` if you're using ESM or TypeScript respectively). This command will send a message, and then edit it with the bot's WebSocket latency.

Your folder structure should now look something like:

```bash {4-5}
├── node_modules
├── package.json
└── src
    ├── commands
    │   └── ping.js
    └── index.js
```

With the file created, we can start writing our commands. Commands in Sapphire need to extend the [`Command`](../../Documentation/api-framework/classes/Command) class.

That all said, here is an example of a `ping` command:

<Tabs groupId="language-choice">
<TabItem value="javascript" label="JavaScript" default>

```javascript {4-10}
const { Command } = require('@sapphire/framework');

module.exports = class PingCommand extends Command {
  constructor(context, options) {
    super(context, {
      ...options,
      aliases: ['pong'],
      description: 'ping pong'
    });
  }
};
```

</TabItem>
<TabItem value="ESM" label="ESM">


</TabItem>
<TabItem value="typescript" label="TypeScript">

```typescript {4-7}
import { ApplyOptions } from '@sapphire/decorators';
import { Command, CommandOptions } from '@sapphire/framework';

@ApplyOptions<CommandOptions>({
  aliases: ['pong'],
  description: 'ping pong'
})
export class PingCommand extends Command {}
```

</TabItem>
</Tabs>

Let's go over what is defined in the constructor in this code:

- `context`: an object that contains file metadata required by the `Piece` class (which `Command` extends) in order to function.
- `name`: by default, the name of the file without the extension, i.e. `ping.js` becomes `ping`, so there's no need to define it.
- `aliases`: other ways users can call the command. You can have as many as you want!
- `description`: some text that you can use to display when a "help" command is used.

There are many other properties available. The full list can be viewed at the [CommandOptions](../../Documentation/api-framework/interfaces/CommandOptions) interface. They will also be explained in upcoming sections.

## Creating the `messageRun` method

Commands have a `messageRun` method, which executes the command logic. Define this below the command's constructor:

<Tabs groupId="language-choice">
<TabItem value="javascript" label="JavaScript" default>

```javascript {8-16}
const { Command } = require('@sapphire/framework');

module.exports = class PingCommand extends Command {
  constructor(context, options) {
    // ...
  }

  async messageRun(message) {
    const msg = await message.channel.send('Ping?');

    const content = `Pong from JavaScript! Bot Latency ${Math.round(this.container.client.ws.ping)}ms. API Latency ${
      msg.createdTimestamp - message.createdTimestamp
    }ms.`;

    return msg.edit(content);
  }
};
```

Any discord.js code can be executed here since the Sapphire framework is an extension of it. The command can be triggered with `@bot ping` or `@bot pong`.
